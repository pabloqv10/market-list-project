Proyecto

A continuación se listan los items más relevantes del proyecto.

  • Idea

    Manejo de lista de mercado y finanzas

  • Descripción del problema

    Cada vez que vamos a hacer supermercado nos encontramos con dos grandes problemas:

    • No recordamos exactamente cuales son los productos que nos hacen falta.
    • No tenemos conocimiento de un histórico de precios relacionado al producto y no sabemos si estamos perdiendo dinero.

  • Justificación

    Partiendo de la necesidad de realizar las compras de suministros para el hogar habitualmente, identificamos que muchas veces es difícil tener un histórico detallado de precios contra los cuales podamos hacer una comparación de los precios actuales e identificar cuáles productos incrementaron, disminuyeron o en caso de cambiar de proveedor, que producto viene siendo mas barato o mas costoso. Tomando en cuenta lo anterior, proponemos diseñar una aplicación la cual nos permita:

    • Crear y almacenar listas de compras.
    • Llevar un histórico detallado de precios, producto a producto.
    • Hacer fácilmente comparaciones entre precios y que nos indique fecha y proveedor del mejor precio, por cada producto.

  • Historia del arte

    A pesar de que ya existen varias aplicaciones para el control de finanzas, comparación de precios de productos entre tiendas y creación de listas, no existe ninguna aun como la que estamos planteando, que combine los 3 componentes y permita almacenar información sobre el tiempo. Durante el desarrollo del proyecto investigaremos más a fondo diferentes ejemplos de estas aplicaciones para tomar un referente de hacia dónde llevar nuestro desarrollo.

  • Objetivos

    • Crear y almacenar listas de compras.
    • Diseñar una aplicación web que permita crear y guardar una o varias listas de compras, histórico de precios, creación de productos, etc.
    • Implementar toda la infraestructura necesaria para el funcionamiento de la aplicación que se pretende desarrollar.
